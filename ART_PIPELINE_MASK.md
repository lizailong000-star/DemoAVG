# ART_PIPELINE_MASK.md

> 任务版本：v3.0e-mask-pipeline-pre（2026-06-04）
> 作用：定义场景 Hotspot Mask 图制作规范，供 image2 / 美术 / 老李 / OpenClaw 共享。

---

## 1. 目的

Mask 图用于：

- **标定** 场景里所有可探索热点的位置和范围（不再用手工 hotspot_data 写死坐标）
- **绑定** 提示框 / glow 高亮 / 范围检测，全部跟实际美术对齐
- **离线解析** 通过工具脚本自动转成 hotspot_data，零手调

本轮（v3.0e-mask-pipeline-pre）只做**离线解析**和**规范文档**，不接运行时。

---

## 2. 文件位置和命名

```
game/images/scene_<场景目录名>/
  bg_far_<场景>.png
  bg_mid_<场景>.png
  bg_near_<场景>.png
  bg_fore_<场景>.png
  hotspots_<场景>.png       ← 本规范定义
```

**当前阶段示例**（养老院大门）：

```
game/images/scene_yangloiyuan/
  bg_far_yangloiyuan.png
  bg_mid_yangloiyuan.png
  bg_near_yangloiyuan.png   (2089×753)
  bg_fore_yangloiyuan.png
  hotspots_yangloiyuan.png  ← 必须 2089×753，跟 bg_near 同尺寸
```

> ⚠️ **目录名拼写说明**：当前阶段沿用旧拼写 ``scene_yangloiyuan``。V2 资源（3200×720）正式重做时统一改为 ``scene_yanglaoyuan``，不在本轮处理。

## 3. Mask 文件规格

| 项 | 规则 |
|---|---|
| 格式 | PNG / RGBA（4 通道带 alpha） |
| 尺寸 | **必须跟同场景 bg_near 完全一致** |
| 坐标基准 | 跟 bg_near 同一套图像坐标 |
| 视差 | 本轮**只做离线解析**，不谈视差；未来运行时显示再讨论 |

**当前 bg_near = 2089×753 → mask 必须 2089×753。**

> ⚠️ V2 升级时 bg_near 改 3200×720，mask 也同步改 3200×720。**两者必须始终同尺寸。**

---

## 4. 像素规则

| 区域 | 像素要求 |
|---|---|
| 非热点区域 | alpha = 0（完全透明） |
| 热点区域 | alpha = 255 + 纯色色块 |

**严禁**：

1. ❌ 渐变
2. ❌ 描边
3. ❌ 半透明
4. ❌ 抗锯齿（必须硬边）
5. ❌ 写文字
6. ❌ 保留背景图
7. ❌ 同一颜色对应多个热点

---

## 5. 颜色映射

每个场景维护一套独立映射。当前**养老院大门**映射：

| 颜色 | Hex | 热点 ID | 说明 |
|---|---|---|---|
| 🔴 红 | ``#FF0000`` | ``gate`` | 养老院大门 |
| 🟢 绿 | ``#00FF00`` | ``notice_board`` | 公告栏 |
| 🔵 蓝 | ``#0000FF`` | ``security_room`` | 保安室 |
| 🟡 黄 | ``#FFFF00`` | ``to_lobby`` | 进入大厅入口 |

未来扩展热点：在该场景的颜色映射里追加一个新色 + 一个新 ID。

---

## 6. 色块形状和位置规则

| 项 | 规则 |
|---|---|
| 形状 | 不限矩形，可以是任意形状（圆/弧/不规则） |
| 大小 | 覆盖物体可交互范围（一般是物体本身 + 周围 20~40px） |
| **底部** | **必须延伸到玩家可站立地面线附近**（保证玩家走到物体前能触发） |
| 顶部 | 留出空间给提示框（建议色块顶部上方至少 60px 留白） |
| 互不重叠 | 不同颜色色块**不允许重叠** |

---

## 7. 解析输出（工具会从 mask 提取的字段）

每个色块解析出：

| 字段 | 含义 |
|---|---|
| ``id`` | 颜色映射对应的热点 ID |
| ``color_hex`` | 色块颜色 |
| ``bbox`` | (x1, y1, x2, y2) 色块外接矩形（mask 图像坐标） |
| ``center_x`` | 色块重心 x（mask 图像坐标，用于提示框对齐） |
| ``center_y`` | 色块重心 y（mask 图像坐标，用于提示框对齐） |
| ``top_y`` | 色块顶部 y（提示框锚点参考） |
| ``bottom_y`` | 色块底部 y（玩家"地面线"参考） |
| ``area_px`` | 色块像素总面积（用于校验是否太小） |

## 8. 工作流（image2 / 老李用）

1. 在 Photoshop / Krita / Procreate 等工具里打开 ``bg_near_<场景>.png`` 作为参考底图
2. 新建一个透明图层
3. 用油漆桶 / 矩形/圆形选区 + 填色，按颜色映射给每个热点画色块
4. **底部贴到玩家可站立地面线**；**顶部留 ≥60px 空白**
5. **隐藏底图层**，**只导出色块图层**为 PNG
6. 文件名按命名规则保存到对应场景目录
7. 跑解析工具校验：``python tools/extract_hotspots_from_mask.py <mask_path>``

## 9. AI 生成 Prompt 模板（给 image2）

```
请基于参考底图 bg_near_<场景>.png，生成一张同尺寸（如 2089×753）的透明 PNG，作为 hotspot mask。

要求：
- 4 通道 RGBA
- 非热点区域 alpha = 0（完全透明）
- 热点区域 alpha = 255 纯色色块（硬边、无抗锯齿、无渐变）
- 颜色映射：
    #FF0000 红 = 大门
    #00FF00 绿 = 公告栏
    #0000FF 蓝 = 保安室
    #FFFF00 黄 = 进入大厅
- 色块覆盖物体可交互范围（物体本身 + 周围 20~40px）
- 色块底部贴到玩家可站立地面线
- 色块互不重叠
- 不写文字、不画描边、不保留底图
```

---

## 10. 校验流程

跑解析工具会输出：

- ✅ 每个色块的 bbox / center / area
- ⚠️ 警告：色块面积过小（< 500px²）
- ⚠️ 警告：色块跟其他色块重叠
- ⚠️ 警告：mask 含有未知颜色（不在映射表里）
- ❌ 错误：尺寸跟 bg_near 不一致
- ❌ 错误：mask 不是 RGBA（缺 alpha 通道）

输出文件：

- ``tools/output/hotspots_<场景>_report.json`` 解析数据
- ``tools/output/hotspots_<场景>_debug.png`` 调试图（原图 + bbox + label）
